1) What is class? 2) What is an instance of a class? 3) thus far we have compared the class to a blueprint. Can you think of another analogy to explain classes? Answer 1:

A class is a like a blueprint of a building. We can use classes to initilize or create space in memory for multiple instances (or objects) and we can use them to do things that are defined in that class.

Answer 2:

An instance of a class is like taking the core properties of an object and use them according to our wishes. We can even change its properties by making changes to code in folder which the class is stored.

Answer 3:

Another analogy that I think of would be compare it to any kind of structure. We can just take any structure and apply it on different areas.
